Settings

System-wide configuration for branding, companies, workflows, financial periods, and global behaviour.

Open Settings near the bottom of the left sidebar (above User Guide). Each sub-item is documented below.

General Settings

Settings → General Settings

Controls the public-facing front page and general application branding:

  • Company title and logo — displayed on the login page and public careers site.
  • About us / short description — text shown on the external job portal homepage.
  • Meta tags — SEO title, description, and social sharing (Open Graph) settings for the public site.
  • Favicon — browser tab icon for the application.

Use this screen when rebranding or updating what external candidates and visitors see before logging in.

Companies

Settings → Companies

Manage the list of legal entities or business units in a multi-company deployment:

  • Add company — register a new company with name, code, and contact details.
  • Edit company — update company profile information.
  • Company switcher — users with access to multiple companies select the active company from the top bar (e.g. "All Companies (SuperAdmin)").

Employee records, payroll runs, and reports can be scoped to a single company or aggregated across companies depending on user permissions.

Company Settings

Settings → Company Settings

Per-company HR and operational defaults that differ between entities:

  • Working hours and timezone — local business rules for the selected company.
  • Default leave group — leave policy applied to new employees in this company.
  • Payroll defaults — currency, pay frequency references, and company-specific payroll rules.
  • Address and contact — used on payslips, letters, and statutory reports.
  • Registration numbers — employer PIN, NSSF number, and other statutory identifiers for filings.
Tip Configure Company Settings immediately after adding a new company and before onboarding employees into that entity.

Print Head Settings

Settings → Print Head Settings

Define the letterhead and header/footer layout for printed and PDF documents:

  • Company logo on printouts — appears on payslips, P9 forms, and HR letters.
  • Header text — company name, address block, and tagline.
  • Footer text — copyright, confidentiality notice, or page numbers.
  • Report-specific layouts — separate print heads for payroll vs. HR reports if configured.

Preview a sample payslip or report after saving to confirm alignment and image quality.

Approval Workflows

Settings → Approval Workflows

Define multi-step approval chains used across the system. Each workflow specifies ordered approver roles or individuals for a transaction type.

Common workflow types include:

  • Leave applications — e.g. Supervisor → HOD → HR.
  • Loan applications — e.g. Supervisor → Finance → HR.
  • Payroll processing — e.g. Payroll Officer → Finance Manager → CEO.
  • Job requisitions — hiring approval chain before posting vacancies.
  • Overtime requests — supervisor approval before payroll inclusion.
  1. Create a new workflow and name it clearly (e.g. "Standard Leave Approval").
  2. Add approval steps in order — each step maps to a role or specific user.
  3. Assign the workflow to the relevant module or leave/loan type in module settings.
  4. Test with a non-production user account to confirm approvers receive items in ESS → Approvals.
Important Changing an active workflow does not retroactively alter items already in the approval queue. Plan workflow changes during quiet periods.

Financial Year Settings

Settings → Financial Year Settings

Define the organisation's fiscal calendar displayed in the top bar (e.g. Financial Year: 01 Jan – 31 Dec, 2026):

  • Start date — first day of the financial year.
  • End date — last day of the financial year.
  • Active year — only one financial year is active at a time for reporting.
  • Year rollover — create the next financial year before the current one closes.

Financial year boundaries affect leave rollover, performance review periods, budget reports, and year-end payroll summaries.

System Settings

Settings → System Settings

Global toggles and integration options that affect the entire application:

  • Authentication options — password login, Google SSO, Azure AD (Microsoft) login enable/disable.
  • Two-factor authentication (2FA) — require OTP on login when enabled.
  • Password policy — expiry days, complexity rules, forced change on first login.
  • Email / SMS notifications — SMTP settings and SMS gateway for OTP and alerts.
  • Biometric integration — connection settings for attendance devices.
  • License and environment — application licence validation and environment-specific flags.
  • Feature toggles — enable or disable optional modules per deployment.
System admin only Incorrect System Settings can lock out users or stop notifications. Restrict access to trusted administrators.

Recommended setup order

  1. Companies — register all entities.
  2. Company Settings — configure each company.
  3. Financial Year Settings — set the active year.
  4. Approval Workflows — define approval chains.
  5. Print Head Settings — set up document branding.
  6. General Settings — configure the public portal.
  7. System Settings — authentication, notifications, integrations.
