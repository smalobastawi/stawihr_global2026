Administration

Manage users, roles, permissions, and audit activity across StawiHR.

Users

Administration → Users — create and manage user accounts linked to employee records.

  1. Click Add User and link to an existing employee or create credentials.
  2. Assign one or more roles that define what the user can access.
  3. Set status to Active or deactivate users who leave the organization.
  4. Use Active / Inactive filters to manage the user list.

Roles & permissions

  • Roles — define role names (e.g. HR Officer, Payroll Admin, Employee).
  • Role Permissions — tick menu and action permissions for each role. Changes apply to all users with that role.
  • Company Permissions — restrict users to specific companies or branches when multi-company mode is enabled.
Principle of least privilege Grant only the permissions needed for each role. Over-permissioning increases security and data exposure risk.

Change password & activity logs

  • Change Password — users update their own password; admins can trigger password reset emails.
  • Activity Logs — review who did what and when for audit and troubleshooting.

See also Policy Documents for publishing HR policies to employees.
