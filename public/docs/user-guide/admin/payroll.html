Payroll

Configure pay components, process monthly payroll, manage loans, handle multi-currency disbursements, and generate statutory reports.

Payroll setup

Under Payroll → Setup configure:

  • PAYE Setup — tax bands and rates.
  • Earning Types — basic salary, allowances, bonuses.
  • Deduction Types — voluntary and statutory deductions.
  • Pay Period — monthly/weekly pay cycles.
  • Pension Schemes — retirement contribution rules.
  • Banks Setup — banks for salary disbursement files.

Processing payroll

  1. Ensure all employee Payroll Profiles are complete with bank details and earnings.
  2. Open Payroll Dashboard for an overview of the current period status.
  3. Go to Process Payroll, select the pay period, and run validation.
  4. Review calculated amounts, resolve errors, and submit for approval if required.
  5. After approval, generate Payment Reports and statutory filings.

Loan management

Payroll → Loan Management includes dashboard, all loans, create loan, applications, pending approvals, manual deductions, loan types, and loan reports.

Employee earnings & deductions

  • Employee Earnings — bulk or individual recurring earnings.
  • Employee Deductions — assign deductions to employees.
  • Payroll Overtimes — include approved overtime in payroll runs.

Payroll reports

Key reports under Payroll → Reports:

  • Payment Reports Dashboard, Salary Change History, P9, NSSF, SHIF, AHL, PAYE.
  • Earnings report, Deductions report, Variance report.

Multi-currency payroll

StawiHR supports employers who must calculate statutory payroll in a local base currency (for example RWF in Rwanda) while paying some employees in a foreign payment currency (for example USD). Local-currency employees are unaffected — the workflow is identical to standard payroll.

Two currencies, two purposes
  • Payroll base currency — used for PAYE, pension, social security, and all statutory returns.
  • Payment currency — the currency sent to the employee’s bank account.

Company currency settings

Configure under Settings → Companies → Edit Company (payroll section):

SettingPurpose
Payroll countryDetermines statutory rules (PAYE bands, pension, etc.)
Payroll base currencyStatutory calculation currency (e.g. RWF, KES)
Default payment currencyDefault bank payment currency when not set per employee
Allow employee payment currencyEnables per-employee payment currency overrides
Exchange rate effective date policyWhich date selects the rate during payroll (period end, start, payment date, or latest on/before period end)

Employee currency settings

Under Payroll → Employees → Create/Edit Employee Payroll:

  • Salary currency — currency of the contract basic salary (e.g. USD).
  • Payment currency — currency paid to the bank (leave blank to use company default).
  • Bank payment currency — optional override for bank upload file labels.

Payment currency fields appear only when Allow employee payment currency is enabled at company level.

Exchange rates

Manage rates under Payroll → Settings → Exchange Rates.

  1. Before each payroll run, enter rates for required currency pairs (e.g. USD → RWF and RWF → USD).
  2. Set the rate, effective date, and optional payroll period.
  3. Click Save Rate — the rate is available immediately for payroll.
  4. After a rate is used in processed payroll, it is locked and cannot be edited (historical payslip integrity).

Processing and validation

When payroll runs for a foreign-currency employee, the system:

  1. Converts salary and fixed foreign-currency earnings to the base currency.
  2. Calculates all statutory deductions in the base currency only.
  3. Converts net pay to the employee’s payment currency for bank disbursement.
  4. Saves both currency sets and the exchange rate snapshot on the payslip record.

If exchange rates are missing, the Process Payroll screen shows a warning listing affected employees. Payroll for those employees cannot complete until rates are entered.

Payslips and bank files

  • Payslips — show statutory amounts in base currency; when payment currency differs, an additional section shows amount paid, exchange rate, and effective date.
  • Statutory reports (PAYE, pension, etc.) — always use base currency amounts only.
  • Bank upload — uses net pay in payment currency; generate separate files per payment currency when your bank requires it.
Recommended monthly workflow Enter exchange rates before payroll cut-off → review rate warnings on Process Payroll → verify payslips show correct base and payment amounts → generate bank files by currency → submit statutory returns in base currency only.
